# Morse 3D 算法逻辑说明文档 (v3.1)

本系统是一个基于离散莫尔斯理论 (Discrete Morse Theory, DMT) 的三维拓扑特征分析引擎。该引擎能够从 3D 概率图中提取拓扑持久性 (Topological Persistence)，识别出图像中的关键拓扑特征（如连通分量、环、空洞及其持久性）。

## 算法概况

`morse_3d.cpp` 实现了四阶段的 DMT 处理流水线，核心目标是将复杂的体素（Voxel）网格转化为轻量化的莫尔斯复形（Morse Complex），并通过矩阵化简（Matrix Reduction）提取拓扑对。

---

## 核心阶段详解

### 第 1 阶段：网格初始化与过滤排序 (Grid Init & Sorting)

1.  **复形构建**：将输入的 $N \times M \times L$ 体素网格扩展为 $(2N+1) \times (2M+1) \times (2L+1)$ 的胞腔复形（Cell Complex）。
    -   **0-cell** (顶点): 位于 $(2i, 2j, 2k)$。
    -   **1-cell** (棱): 两个顶点之间的连接。
    -   **2-cell** (面): 由棱围成的面。
    -   **3-cell** (体): 最小的体元单元。
2.  **属性分配**：
    -   `dims`: 记录每个 cell 的维度（0, 1, 2, 3）。
    -   `keys`: 为每个 cell 分配标量值（取其相邻体素的最大值），用于确定过滤顺序（Filtration Order）。
3.  **并行排序**：使用 `std::execution::par_unseq` 进行大规模并行排序，确定胞腔加入复形的先后顺序。

### 第 2 阶段：离散梯度场配对 (Discrete Gradient Pairing)

1.  **配对搜索**：尝试将维度为 $d$ 的胞腔与紧邻的维度为 $d+1$ 的胞腔配对，形成**离散向量场**。
2.  **规则**：一个胞腔最多参与一个配对，且配对必须满足离散莫尔斯理论的正则性要求。
3.  **关键胞腔 (Critical Cells)**：
    -   无法参与配对的胞腔被定义为**关键胞腔**。
    -   关键胞腔的维度对应其拓扑意义（Index 0: 极大值/连通分量, Index 1/2: 鞍点, Index 3: 极小值/空洞）。

### 第 3 阶段：V-路径 搜索与莫尔斯边界 (V-Path DFS)

1.  **离散流线**：沿着由离散梯度场定义的路径（V-Path）进行追踪。
2.  **莫尔斯边界构建**：
    -   使用**带记忆的深度优先搜索 (Memoized DFS)** 寻找从一个关键胞腔出发，沿梯度路径可达的其他关键胞腔。
    -   这些连接定义了莫尔斯复形（Morse Complex）的边界关系，极大地减少了后续计算的网格规模。
3.  **内存优化**：在构建完边界矩阵后立即清理 DFS 缓存，以防止处理超大规模 3D 数据时出现 OOM (Out of Memory)。

### 第 4 阶段：持久性化简 (Reduction & Persistence)

1.  **边界矩阵化简**：在莫尔斯复形上执行标准持久同调化简算法。
2.  **拓扑对配对**：
    -   识别 **Birth-Death** 对：一个拓扑特征何时产生（Birth），何时因更高维度的特征填充而消失（Death）。
    -   **持久性 (Persistence)** = $|Birth - Death|$。
3.  **异常处理**：
    -   **Ghost Pair 过滤**：设定 $10^{-9}$ 的容差 epsilon，过滤由于浮点精度造成的极细微噪声。
    -   **Essential $H_0$**：记录永远不会死亡的连通分量（如全局最高背景）。

## 性能优化与技术指标

| 优化点 | 实现方式 | 作用 |
| :--- | :--- | :--- |
| **并行计算** | OpenMP + Parallel STL | 利用多核 CPU 加速排序与初始属性计算。 |
| **硬件断路器** | `int32_t` 溢出检查 | 硬性限制网格规模（< 21.4 亿单元），防止索引溢出导致崩溃。 |
| **内存管理** | `memo.clear()` | 在化简前释放 V-Path 缓存，优化内存占用曲线。 |
| **数据紧凑性** | `StaticVec` (栈分配) | 使用固定大小数组代替 `std::vector` 处理临近点，减少堆分配开销。 |

## 输入输出定义

-   **输入**：3D NumPy 数组（通常为 Vessel 分割或概率热图）。
-   **输出** (Python Dict)：
    -   `births / deaths`: 拓扑对的值。
    -   `dims`: 对应的维度（0, 1, 2）。
    -   `birth_coords / death_coords`: 特征在三维空间中的归一化坐标。
